# TableLore v1.0 Release Notes

TableLore v1.0 ships the first usable skill-first release for practical data analysis hygiene in AI coding agents.

## Included

- Codex/Claude-compatible `tablelore/SKILL.md` with local-first analysis workflow.
- Reference guidance for profiling, engine selection, notebook/script hygiene, modeling guardrails, and examples.
- Optional local CSV/TSV profiling helper at `tablelore/scripts/table_profile.py`.
- Fixture-backed tests for the profiling helper.
- MIT license.
- Release checklist at `docs/releases/v1.0-checklist.md`.

## Boundaries

TableLore v1.0 is not AutoML, not a hosted reporting service, and not a packaged CLI. The helper script is intentionally small and local-first. It is meant to support the skill workflow, not replace project-specific analysis.

## Verification

Run:

```bash
python3 -m pytest tests -q
python3 tablelore/scripts/table_profile.py tests/fixtures/customer_churn_sample.csv --target churned
```

## Known Gaps

- The helper currently supports CSV and TSV only.
- DuckDB, Polars, Excel, JSON/JSONL, SQLite, and DuckDB-file workflows are documented as agent guidance but are not implemented in the helper.
- Full `tablelore profile` CLI packaging is deferred to the v2 backlog.
- Dogfood evidence is committed under `dogfood/com47/`.
