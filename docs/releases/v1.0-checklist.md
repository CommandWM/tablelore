# TableLore v1.0 Release Checklist

## Scope

- [x] Skill-first release, not a packaged Python CLI.
- [x] Concise `tablelore/SKILL.md` with trigger description and default workflow.
- [x] Profiling checklist reference.
- [x] Engine selection reference.
- [x] Notebook and script hygiene reference.
- [x] Modeling guardrails reference.
- [x] Example prompts and expected response shapes.
- [x] Optional local CSV/TSV profiling helper.
- [x] Fixture-backed tests.
- [x] MIT license.

## Verification Commands

Run before tagging or merging v1:

```bash
python3 /Users/matthewdavis/.codex/skills/.system/skill-creator/scripts/quick_validate.py tablelore
python3 -m pytest tests -q
```

## Release Boundaries

- No AutoML.
- No hosted services.
- No database credential management.
- No full `tablelore profile` package entry point.
- No external data upload by default.

## Follow-Up Validation

- Dogfood on a real unfamiliar CSV analysis.
- Dogfood on a Parquet or multi-file DuckDB-style inspection.
- Dogfood on a notebook deliverable.
- Dogfood on a messy-notebook-to-script conversion.
- Dogfood on a modeling request with leakage checks.

These dogfood runs should create concrete evidence before expanding helper scope.
